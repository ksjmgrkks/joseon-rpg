# CLAUDE.md — 조선시대 도트 횡스크롤 RPG

> 이 파일은 Claude Code가 매 세션 시작 시 자동으로 읽는 영구 컨텍스트입니다.
> **사용법:** 프로젝트 루트에 `CLAUDE.md`로 저장하세요. `[채워넣기]` 부분은 실제 값으로 교체.
> 폰/텔레그램 세션에서 짧게 지시해도 Claude가 헛다리를 덜 짚도록, 구조와 규칙을 최대한 구체적으로 적어두는 게 핵심입니다.

## 프로젝트 개요
- **장르:** 조선시대 배경 도트(픽셀아트) 횡스크롤 액션 RPG
- **엔진:** Godot 4.x — **GDScript 전용** (C# 금지: web export가 안 됨)
- **타깃 플랫폼:** [PC/Steam, 모바일, 웹 중 선택]
- **시대 배경:** [조선 초/중/말 — 선택]
- **톤:** [진지 / 코믹 / 괴담 등]

## 개발 환경 (중요)
- 주 개발은 **폰 + 텔레그램 + Claude Code**로 이루어진다. 나는 폰에서 Godot 에디터나 게임 화면을 볼 수 없다.
- 따라서 **시각 확인이 필요한 작업(픽셀 정렬, 애니메이션 타이밍, UI 레이아웃, 손맛)은 절대 "다 됐다"고 단정하지 말 것.** 대신 데스크톱에서 확인이 필요하다고 알려줄 것.
- 로직·시스템·리팩터링·버그픽스 위주로 작업한다.

## 절대 규칙 (Always)
1. **작업 단위마다 git 커밋.** 의미 있는 변경 후 명확한 메시지로 커밋한다. 큰 변경 전엔 현재 상태를 먼저 커밋.
2. **GDScript만 사용.** C#/.NET 코드 생성 금지.
3. **기존 파일을 수정/삭제하기 전에 먼저 보여주고 확인받기.** 폰에서는 내가 코드를 꼼꼼히 못 읽으니, 파괴적 변경은 반드시 사전 확인.
4. **한 번에 하나의 작업만.** 여러 시스템을 동시에 건드리지 말 것. 요청이 크면 단계로 쪼개서 제안부터.
5. **로직 변경 시 헤드리스 테스트 제공.** 가능하면 `godot --headless`로 돌려 통과/실패 결과를 텍스트로 보고.

## 절대 금지 (Never)
- 존재하지 않는 노드·시그널·API를 추측해서 쓰지 말 것. 불확실하면 프로젝트 파일을 먼저 읽고 확인.
- 한 번에 게임 전체를 만들려 하지 말 것.
- 시각적 결과를 안 보고 "완성됐다"고 보고하지 말 것.

## 자주 쓰는 명령 (web export 시각 루프)
```bash
# 웹 빌드 (폰 브라우저로 결과 확인용)
godot --headless --export-release "Web" build/index.html

# 헤드리스로 특정 테스트 씬 실행
godot --headless res://tests/[테스트씬].tscn
```
> 빌드 산출물을 [정적 호스팅 위치]에 올린 뒤 폰 브라우저로 확인하는 것이 기본 피드백 루프.
