# CLAUDE.md — 조선시대 도트 횡스크롤 RPG

> 이 파일은 Claude Code가 매 세션 시작 시 자동으로 읽는 영구 컨텍스트입니다.
> **사용법:** 프로젝트 루트에 `CLAUDE.md`로 저장하세요. `[채워넣기]` 부분은 실제 값으로 교체.
> 폰/텔레그램 세션에서 짧게 지시해도 Claude가 헛다리를 덜 짚도록, 구조와 규칙을 최대한 구체적으로 적어두는 게 핵심입니다.

## 프로젝트 개요
- **제목:** 「호환기담」(虎患奇譚)
- **장르:** 조선시대 배경 도트(픽셀아트) 횡스크롤 액션 RPG
- **엔진:** **Godot 4.6.3 고정** — **GDScript 전용** (C# 금지: web export가 안 됨)
- **타깃 플랫폼:** 모바일(우선) + 웹 미리보기(HTML5)
- **시대 배경:** 가상의 조선 후기풍 산골 (2026-06-11 확정)
- **톤:** 은은한 괴담 사극 — 수묵·한지의 절제된 색, 민담 정서, 과한 공포 연출 금지 (2026-06-11 확정)

## 개발 환경 (중요)
- 폰(텔레그램) 세션과 PC 세션이 섞인다. PC 세션에서는 Godot 윈도우 실행·스크린샷 캡처가 가능하다.
- **시각 검증은 스크린샷 캡처 루프로 스스로 확인**한다 (tools/ 의 캡처 러너). 캡처로 확인 못 한 것(애니메이션 타이밍·손맛)만 사용자 확인 항목으로 남길 것.
- 로컬 Godot: `C:\Users\User\Downloads\Godot_v4.6.3-stable_win64.exe\Godot_v4.6.3-stable_win64.exe` (본체 exe 사용, console 래퍼 금지, 항상 timeout 감싸기)

## 절대 규칙 (Always)
1. **작업 단위마다 git 커밋.** 의미 있는 변경 후 명확한 메시지로 커밋한다. 커밋이 곧 롤백 지점 — 확인 대신 커밋으로 통제한다.
2. **GDScript만 사용.** C#/.NET 코드 생성 금지.
3. **자율 작업 모드 (2026-06-11 사용자 승인):** 되돌릴 수 있는 변경(파일 수정/생성/테스트/커밋)은 사전 확인 없이 진행. 단, 푸시 불가능한 파괴(히스토리 재작성 등)와 외부 공개만 확인받기.
4. **다 만들고 수정해나가는 방식.** 큰 그림 먼저 완성 → 스크린샷·테스트 보고 → 다듬기.
5. **로직 변경 시 헤드리스 테스트, 비주얼 변경 시 스크린샷 캡처**로 자체 검증하고 결과를 있는 그대로 보고.

## 절대 금지 (Never)
- 존재하지 않는 노드·시그널·API를 추측해서 쓰지 말 것. 불확실하면 프로젝트 파일을 먼저 읽고 확인.
- 검증(테스트/스크린샷) 없이 "완성됐다"고 보고하지 말 것. 미감 최종 판단은 사용자 몫으로 명시.

## 프로젝트 구조
```
res://
├── scenes/        # .tscn 씬 파일
│   ├── player/
│   ├── enemies/
│   ├── levels/
│   └── ui/
├── scripts/       # .gd 스크립트
├── assets/
│   ├── sprites/   # 캐릭터·적·오브젝트 스프라이트
│   ├── tilesets/  # 맵 타일셋
│   ├── fonts/     # 한글 픽셀 폰트
│   └── audio/
├── tests/         # 헤드리스 테스트 씬
└── CLAUDE.md
```
> 실제 구조가 다르면 위를 실제에 맞게 수정하세요. Claude가 이걸 지도로 삼습니다.

## 아트/렌더 규칙 (코드가 지켜야 할 값)
- **캐릭터 높이:** 48px
- **타일 크기:** 32×32
- **팔레트:** 제한 팔레트 24~32색, 단청·먹·한지 톤. 새 색을 임의로 추가하지 말 것. (구체 컬러 코드는 `docs/STYLE_BIBLE.md` 에서 잠금 — 작성 예정)
- **카메라:** 픽셀 퍼펙트. 텍스처 필터는 Nearest. 서브픽셀 위치 보간 주의.
- **렌더 배율:** 정수배 스케일만 (1×, 2×, 3× …)

## 코드 컨벤션
- 노드 이름: PascalCase (예: `PlayerCharacter`)
- 변수/함수: snake_case (GDScript 관례)
- 시그널 연결은 **코드 방식**(`signal_name.connect(callable)`)으로 통일 — 에디터 우클릭 연결은 추적이 어려움
- 새 시스템은 작은 단일 책임 스크립트로 분리

## 자주 쓰는 명령 (web export 시각 루프)
```bash
# 웹 빌드 (폰 브라우저로 결과 확인용)
godot --headless --export-release "Web" build/index.html

# 헤드리스로 특정 테스트 씬 실행
godot --headless res://tests/[테스트씬].tscn
```
> 빌드 산출물을 **GitHub Pages 또는 Netlify**(Phase 1 web export 단계에서 확정)에 올린 뒤 폰 브라우저로 확인하는 것이 기본 피드백 루프.
